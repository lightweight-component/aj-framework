选型原则：简明快速、小巧精致、开源主流。

框架选用

  • Win32/CentOS
  • Java 版本：要求 Java 1.7 或以上，请注意不能是 OpenJDK。
  • Tomcat 7 或以上应用服务器，最低兼容 Servlet 3.0 规范

开发工具

  • Eclipse 4.0 或以上。推荐插件 Eclipse Color（改变主题颜色）、LESS 编辑器（修改样式用）
  • SVN 命令行或 Subclipse
  • 文本编辑器推荐 Sublime Text 或 Notepad++
  • 浏览器 Firefox（丰富的插件和扩展 FireFTP/FireSSH/SQLite Manger）和 Chrome
  • 本地 HTML 可视化编辑器 BlueGriffon 和 Eclipse 内置的 Web Editor
  • 数据库工具 MySQL Workbench。

Java 组件

日志服务

AJAXJS 针对 JDK 自带的 logger 稍加扩展，使用例子如下： 

private static final LogHelper log = LogHelper.getLog(TestLogHelper.class);
log.warning("fooo");
log.info("bar");

对于 WARNING 级别信息会以磁盘文件方式保存在 /META-INF/logger/log.txt 下。

如果你习惯其他 log，建议使用 slf4j 或 apache common logging，可参考我写的教程《使用 slf4j + Java.util.logger》。

单元测试基于 JUnit 4 + Mock

相关教程请参阅我写的《 JUnit + Mockito 单元测试》系列文章。代码模板如下。

import static org.junit.Assert.*;
import org.junit.Test;

import com.ajaxjs.util.LogHelper;

public class TestLogHelper {
	@Test
	public void testLog() {
		LogHelper log = new LogHelper(TestLogHelper.class);
		assertNotNull(log);
		log.warning("dsada");
		log.info("dsada");
	}


	@org.junit.Test
	public void testCatchEx() {
		LogHelper log = LogHelper.getLog(TestLogHelper.class);
		try {
			throw new Throwable("h");
		} catch (Throwable e) {
			log.warning(e);
		}
	}
}

数据库组件

数据库当前支持 MySQL/SQLite。如果项目并发量不大可使用 SQLite，更简单。相关驱动下载如下。

  • MySQL 驱动
  • SQLite 驱动
ORM：MyBatis

MyBatis 支持 XML、注解和 SQLBuilder 这三种方式构建 SQL 语句。我比较多采用 SQLBuilder，可在 Java 中动态组织 SQL，便于封装。

数据库连接池

使用 Tomcat 自带的 JDBC Pool，支持高并发应用环境，超简单，核心文件只有 8 个。

后端字段验证器

数据验证一般基于 JSR 303 规范。常见的 Hibernate Validator 体积 13MB，另外一个 Apache BVal 才 400kb——果断后者。参见我写的《数据验证框架 Apache BVal 简介》。
