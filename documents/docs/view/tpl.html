HTML 页面

页面既可以静态的，也可以是像 jsp 那样动态的。下面分别是纯 html 页面和一个简单跳转。

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>模版</title>
    <!--宽度 320px -->  
    <meta name="viewport" content="width=320,user-scalable=0,initial-scale=1.0,maximum-scale=1.0, minimum-scale=1.0" />
    <stlye>
      body{margin:0;padding:0;}
    </style>
  </head>
  <body>
  </body>
</html> 
<meta http-equiv="refresh" content="0; url=index.jsp" />

动态页面既可以直接路径访问的 jsp，也可以透过 servlet 动态跳转的 jsp。

<%@page pageEncoding="UTF-8"%>
<%@taglib prefix="commonTag" tagdir="/WEB-INF/tags/public"%>
<!DOCTYPE html>
<html>
	<commonTag:head lessFile="/asset/less/user.less" title="列表">
		<style>
			html, body, table {
				height: 100%;
			}
		</style>
	</commonTag:head>
	<body>
		<table width="100%" height="100%" align="center" valign="middle">
			<tr>
				<td align="center" valign="middle">
					<div class="box">
						<h1>${info.name}</h1>
						<a href="javascript:history.go(-1);">返回列表</a>
						<a href="edit/${info.id}">编辑记录</a>
						<a href="../edit/list">进入编辑列表</a>
					</div>
				</td>
			</tr>
		</table>
                <%@include file="WEB-INF/jsp/public/footer.jsp"%>
     </body>
</html>

建議页面中保留 <%@page pageEncoding="UTF-8"%>，不然 Eclipse 容易打开乱码。<%@include file="/WEB-INF/jsp/public/footer.jsp"%> 为插入文件，建議保存在 /WEB-INF/jsp/ 下。

如果不想 url 后面显示 .jsp 扩展名，可以在 web.xml 指定。

<servlet>  
        <servlet-name>service</servlet-name>  
        <jsp-file>/public/service.jsp</jsp-file>  
</servlet>  
      
<servlet-mapping>  
        <servlet-name>service</servlet-name>  
        <url-pattern>/service/projectcase/*</url-pattern>  
</servlet-mapping>

网页换肤

如果网站结构、样式趋于统一，可以抽象为一模版，那么这样就方便了换肤。如下例就是一个统一界面的模版。

<%@page pageEncoding="UTF-8"%>
<%@taglib prefix="tags" tagdir="/WEB-INF/tags/"%>
<tags:content>
	<jsp:attribute name="body">
		<!-- Editable AREA|START -->	
			<p>密山市贵宾楼建筑安装有限公司</p>
			<p>密山市贵宾家园物业管理有限公司</p>
			<p>密山市交换空间装饰限公司</p>
			<p>密山市贵宾楼服务有限责任公司</p>
			<p>北京江海金苑商务宾馆</p>
			<p>密山兴凯湖大酒店（五星级）</p>
		<!-- Editable AREA|END --> 
	</jsp:attribute>
</tags:content>

如此一来，制作多平台的“响应式”页面也更为容易。既可以服务端根据 ua 判定输出，也可以在前端 CSS 响应式输出。

可编辑页面

不管静态页面还是动态页面，只要插入了下面注释的语句，就可以被后台编辑。
